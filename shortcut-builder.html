🔧 Mini Shortcuts Builder

7 tiny shortcuts — no variable collisions, no bugs

Why 7 mini shortcuts?
One big shortcut with 35 actions has variable name collisions — iOS reuses "Dictionary Value" across all actions and picks the wrong one.

Each mini shortcut has only 5 actions with one single value, so there's nothing to collide. A master shortcut runs all 7 in sequence.
⚙️ Enter your details once

Tap any button to test the API. You should see a single number (e.g., "1014" or "14").

Build 7 Mini Shortcuts

Each one takes ~3 minutes — same 5 steps every time

📋 The Pattern (same for all 7)
Every mini shortcut has exactly 5 actions:
Action 1: Current Date
Add Action → search "Current Date" → tap it
Action 2: Format Date
Format [Current Date] → Custom → yyyy-MM-dd
Action 3: URL (get contents)
Get Contents of URL → paste the URL for this metric
Each shortcut has a unique URL — the copy button below gives you the right one
Action 4: Number
Number → [Contents of URL]
The API returns a plain number as text. This converts it.
Action 5: Log Health Sample
Log [Number] as [TYPE] in [UNIT] on [Current Date]

Shortcut 6: Sync Weight
Logs body weight in kg (only if logged that day)
1. New shortcut → name: "Health - Weight"
2. Current Date → 3. Format Date (yyyy-MM-dd) → 4. Get Contents of URL:
Enter API details above
5. Number → [Contents of URL]
6. If → [Number] is greater than 0
7. (inside If) Log Health Sample → Type: Body Mass · Unit: kg · Value: [Number] · Date: [Current Date]
Otherwise — leave empty
End If
⚠️ The If check prevents logging 0 kg to Health on days you didn't weigh in. The Log action must be INSIDE the If block (indented between If and Otherwise).
